## Prerequisites

### Hardware Requirements
- **GPU**: NVIDIA GPU with CUDA 11.8+ (RTX 3090 or better recommended)
- **CPU**: Intel i7/i9 or AMD Ryzen 7/9 with 8+ cores
- **RAM**: Minimum 32GB, recommended 64GB
- **Storage**: 500GB SSD for models and data

### Software Requirements
- Python 3.10+
- Docker & Docker Compose (for containerized deployment)
- Git
- CUDA Toolkit 11.8
- cuDNN 8.7+

## Local Development Setup

### 1. Clone Repository
```bash
git clone https://github.com/yourname/aerial-threat-detection.git
cd aerial-threat-detection
```

### 2. Create Virtual Environment
```bash
python3.10 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

### 3. Install Dependencies
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Verify CUDA Installation
```bash
python -c "import torch; print(torch.cuda.is_available())"
python -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"
```

### 5. Download Pre-trained Models
```bash
# Download YOLOv8 model
python -m ultralytics.yolo model download yolov8x.pt

# Download Faster R-CNN model
wget https://download.pytorch.org/models/fasterrcnn_resnet50_fpn_coco-258fb6c6.pth -O models/fasterrcnn.pth

# Download Kalman Filter models
wget https://example.com/kalman_filter_model.pth -O models/kalman_filter.pth
```

### 6. Set Up Environment Variables
```bash
cp .env.example .env
# Edit .env with your configuration
```

### 7. Initialize Database
```bash
python -m alembic upgrade head  # If using Alembic migrations
# OR manually run:
psql -U postgres -d threat_detection < init.sql
```

### 8. Run Development Server
```bash
python src/api/main_api.py
# Server will be available at http://localhost:8000
```

## Docker Deployment

### 1. Build Docker Image
```bash
docker build -t aerial-threat-detection:latest .
```

### 2. Run Single Container
```bash
docker run --gpus all \
  -p 8000:8000 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/models:/app/models \
  -e DATABASE_URL=postgresql://user:password@db:5432/threat_detection \
  aerial-threat-detection:latest
```

### 3. Run Full Stack with Docker Compose
```bash
docker-compose up -d
```

### 4. Verify Services
```bash
# Check running containers
docker-compose ps

# View API logs
docker-compose logs -f api

# Test API health
curl http://localhost:8000/health
```

### 5. Stop Services
```bash
docker-compose down
docker-compose down -v  # Remove volumes too
```

## Cloud Deployment

### AWS Deployment with ECS

#### 1. Create ECR Repository
```bash
aws ecr create-repository --repository-name aerial-threat-detection --region us-east-1
```

#### 2. Build and Push Image
```bash
docker build -t aerial-threat-detection:latest .
docker tag aerial-threat-detection:latest <ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com/aerial-threat-detection:latest
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com
docker push <ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com/aerial-threat-detection:latest
```

#### 3. Create ECS Task Definition
```bash
aws ecs register-task-definition --cli-input-json file://ecs-task-definition.json
```

#### 4. Create ECS Service
```bash
aws ecs create-service \
  --cluster threat-detection \
  --service-name aerial-threat-api \
  --task-definition aerial-threat-detection:1 \
  --desired-count 1 \
  --launch-type EC2
```

### GCP Deployment with Cloud Run

```bash
# Authenticate
gcloud auth login

# Build image
gcloud builds submit --tag gcr.io/PROJECT_ID/aerial-threat-detection

# Deploy
gcloud run deploy aerial-threat-detection \
  --image gcr.io/PROJECT_ID/aerial-threat-detection \
  --platform managed \
  --region us-central1 \
  --memory 32Gi
```

### Kubernetes Deployment

```bash
# Create namespace
kubectl create namespace threat-detection

# Create ConfigMap for environment
kubectl create configmap app-config --from-file=configs/ -n threat-detection

# Deploy using Helm
helm install aerial-threat ./helm/aerial-threat-detection -n threat-detection
```

## Configuration

### Environment Variables (.env)
```

REDIS_URL=redis://localhost:6379
LOG_LEVEL=INFO
API_HOST=0.0.0.0
API_PORT=8000
WORKERS=4
DETECTION_CONFIDENCE_THRESHOLD=0.5
TRACKING_HISTORY_SIZE=30
```

```yaml

  yolo_model: yolov8x.pt
  confidence_threshold: 0.5

  prediction_horizon: 30

  map_width: 1000
  map_height: 1000
  replan_threshold: 0.2

  port: 8000
  workers: 4
  log_level: INFO
```

## Troubleshooting

### CUDA Not Found
```bash
# Verify CUDA installation
nvcc --version
nvidia-smi

# Reinstall PyTorch with CUDA support
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
```

### Out of Memory (OOM)
```bash
# Reduce batch size
# Reduce model size (use yolov8m instead of yolov8x)
# Enable model quantization
```

### Database Connection Issues
```bash
# Check PostgreSQL service
sudo service postgresql status

# Test connection
psql -U postgres -h localhost -d threat_detection
```

### Model Loading Errors
```bash
# Verify model paths
ls -la models/

# Re-download models
bash scripts/download_models.sh --force
```
